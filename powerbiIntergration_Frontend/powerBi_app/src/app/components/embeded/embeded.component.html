Power BI Embedded

Power BI Embedded (l'offre Azure) vous permet d'intégrer du contenu Power BI tel que des rapports, des tableaux de bord et des tuiles dans votre application. Pour utiliser votre application, vos utilisateurs n'auront pas besoin de se connecter à Power BI ni d'avoir une licence Power BI.

Prérequis

Avant de commencer ce tutoriel, vérifiez que vous disposez à la fois des dépendances Power BI et de code énumérées ci-dessous :

Dépendances Power BI

  • Votre propre locataire Azure Active Directory.
  • Pour authentifier votre application auprès de Power BI, vous aurez besoin de l'une des options suivantes :
    • Principal de service - Un objet principal de service Azure Active Directory (Azure AD) qui permet à Azure AD d'authentifier votre application.
    • Licence Power BI Pro - Cela sera votre utilisateur principal et votre application l'utilisera pour s'authentifier auprès de Power BI.
    • Une licence Power BI Premium par utilisateur (PPU) - Cela sera votre utilisateur principal et votre application l'utilisera pour s'authentifier auprès de Power BI.

Remarque : Pour passer en production, vous aurez besoin d'une capacité dédiée.

Intégration d'un Rapport Power BI dans une Application Java avec Power BI Embedded

L'intégration d'un rapport Power BI dans une application Java à l'aide de Power BI Embedded est une procédure en plusieurs étapes qui nécessite une configuration minutieuse à la fois dans Azure Active Directory et Power BI. Voici un guide illustré pour vous guider à travers ce processus :

Étape 1 : Configuration dans Azure Active Directory

  1. Créez un Nouveau Répertoire dans Microsoft Azure.
  2. Accédez à Azure Active Directory et sélectionnez Gérer > Utilisateurs. Ajoutez un nouvel utilisateur et attribuez-lui le rôle d'administrateur général.
  3. Connectez-vous à Microsoft avec le nouvel utilisateur créé.
  4. Dans Azure Active Directory, accédez à Gérer > Inscriptions d'applications et créez une nouvelle inscription d'application. Notez l'ID de cette application ainsi que l'ID du répertoire.
  5. Accédez à Gérer > Certificats et secrets pour ajouter un secret à l'application. Conservez cette clé précieusement.
  6. Dans API Permissions, ajoutez les permissions nécessaires pour l'application.
  7. Dans Azure Active Directory, créez un nouveau groupe et ajoutez le nouvel utilisateur en tant que membre.

Étape 2 : Configuration dans Power BI

  1. Connectez-vous à Power BI avec vos identifiants.
  2. Créez un espace de travail (Workspace) et ajoutez vos rapports à cet espace.
  3. Ouvrez le rapport que vous souhaitez intégrer, allez en mode d'intégration et notez l'ID du rapport ainsi que l'ID de l'espace de travail.
